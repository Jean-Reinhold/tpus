Testcontainers

Suba a infraestrutura de verdade no teste

@Testcontainers class PedidoRepositorioIT { @Container static PostgreSQLContainer<?> pg = new PostgreSQLContainer<>("postgres:16"); @Test void persisteERecuperaPedido() { /* driver JDBC real */ } }
Regra prática: mocke o que é seu e já tem teste unitário; containerize o que é dos outros: banco, fila, broker, storage.
  • Postgres, Kafka e Redis reais, em containers descartáveis, um por suíte de teste.
  • O teste fala com o driver e o dialeto de verdade: constraints, transações e tipos incluídos.
  • Mesma versão do banco de produção, sem instalar nada na máquina.
  • Mais caro que mock: use onde o contrato importa, seguindo a pirâmide.
